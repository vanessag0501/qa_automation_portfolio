# OpenCart Manual Testing Project

A full manual QA engagement on a live e-commerce demo storefront, producing the documentation a tester would hand off on a real project.

## Contents
- `test-plan.md` - scope, environment, test types, and entry/exit criteria
- `test-cases.xlsx` - registration and login test cases with steps, expected results, and execution status
- `bug-report.xlsx` - defects found during execution, with severity, priority, and reproduction steps
- `test-summary.md` - execution summary with pass/fail counts and a conclusion

## Target
demo.opencart.com

## Defects found
- Missing Confirm Password field on the registration form
- Validation and error messages render in the upper right corner and are easy to miss

See the repo-level BUGS.md for the full defect log.
